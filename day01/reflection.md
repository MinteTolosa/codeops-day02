# Git Reflection

### 1. What is the difference between Git and GitHub?
Git is a tool that runs locally on my own computer to track changes in my code, save history, and make sure I do not accidentally ruin my files. GitHub is a website on the internet where I upload those saved Git files. It acts as a cloud backup and makes it easy for me to share my exercises with my advisor.

### 2. What is the purpose of the terminal?
The terminal is a text based interface that lets me talk directly to my computer operating system. Instead of clicking on folders and buttons with a mouse, I type precise commands to quickly navigate directorie, run programs like Git, and manage files.

### 3. What does git add do?
It takes the changes I have made to my files and puts them into a temporary staging area. It tells Git exactly which updates or new files I want to include in my next save point. It is like boxing up items before taping the box shut.

### 4. What is the purpose of git commit?
It is takes everything in the staging area and permanently saves it into my repository history as a snapshot. Each commit requires a message, which works like a timestamped note explaining what features or exercises I completed at that exact moment.

### 5. Write the daily Git workflow in the correct order.
This is the standard order I use to save and share my daily exercises:
1. git status
2. git add .
3. git commit -m "Message describing my work"
4. git push origin main
